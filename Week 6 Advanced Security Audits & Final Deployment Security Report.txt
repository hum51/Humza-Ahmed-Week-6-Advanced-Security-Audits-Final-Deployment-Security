2. Secure Deployment Practices:
2.1 Automatic Security Updates:
To ensure the host OS is always patched, we configured unattended-upgrades on the production server.
Unattended-Upgrade::Allowed-Origins {
    "${distro_id}:${distro_codename}";
    "${distro_id}:${distro_codename}-security";
    "${distro_id}ESMApps:${distro_codename}-apps-security";
    "${distro_id}ESM:${distro_codename}-infra-security";
};
Unattended-Upgrade::Automatic-Reboot "true";
Unattended-Upgrade::Automatic-Reboot-Time "03:00";

2.2 Docker Security Best Practices:
We hardened the Docker setup using the following practices:
•	Non-Root User: The container runs as a specific user (UID 10001) instead of root.
RUN addgroup -g 10001 -S appgroup && adduser -u 10001 -S appuser -G appgroup
USER appuser
•	Image Scanning: Integrated Trivy into the CI/CD pipeline.
trivy image --severity HIGH,CRITICAL --exit-code 1 myapp:latest

•	Docker Bench Security: Ran docker-bench-security against the host.
•	Result: 85% pass rate. Failed checks (like "Allow default Docker bridge") were mitigated by using user-defined bridge networks.

•	Read-only Root Filesystem: Containers are run with --read-only to prevent runtime modifications.

3. Final Penetration Testing:
A comprehensive penetration test was performed using Burp Suite Professional and Metasploit targeting the staging environment.
•	Tools: Burp Suite (Repeater, Intruder), Nmap, Metasploit (auxiliary/scanner/http)

3.2 Vulnerability Findings & Remediation:
•	PT-01: Session Cookie Missing HttpOnly Flag 
•	Tool: Burp Suite 
•	Risk: Medium 
•	Description: Session token was accessible via JavaScript, making it vulnerable to XSS theft. 
•	Remediation Applied: Set Secure, HttpOnly, and SameSite=Strict flags in the application server config. 
•	PT-02: Weak TLS Ciphers 
•	Tool: Nmap (ssl-enum-ciphers) 
•	Risk: Low 
•	Description: Server accepted TLSv1.1 and weak ciphers (CBC mode). 
•	Remediation Applied: Updated Nginx config to allow only TLSv1.3. 
•	PT-03: Username Enumeration 
•	Tool: Burp Intruder 
•	Risk: Medium 
•	Description: Login error messages differed ("User not found" vs "Password incorrect"). 
•	Remediation Applied: Changed error message to generic: "Invalid username or password." 
•	PT-04: X-Frame-Options Missing 
•	Tool: Burp Spider 
•	Risk: Info 
•	Description: Clickjacking potential. 
•	Remediation Applied: Added X-Frame-Options: DENY to web server headers.

4. Deployment Architecture:
The application is deployed behind a reverse proxy with the following security layers:
1.	Cloudflare WAF: DDoS protection and IP whitelisting.
2.	Nginx Reverse Proxy: Terminates SSL (Let's Encrypt).
3.	Docker Swarm / Kubernetes: Orchestration with secret management.
4.	PostgreSQL: Isolated in a private subnet; no public exposure.
